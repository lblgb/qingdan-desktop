# `v0.20` 验收清单

## 文件说明

本文档用于沉淀 `v0.20` 的统一手工验收口径，作为本轮实施、回归和收口的唯一执行清单。

## 验收范围

- 任务优先级正式落地并可持久化。
- 左侧 `更多条件` 扩展到任务组、优先级、时间范围和排序方式。
- 列表支持轻量批量操作。
- `任务概览` 保持弹窗形态并增强统计内容。
- 快捷操作强化覆盖全局、列表和任务级高频动作。
- 主界面保持当前产品骨架，不新增独立概览页或重做整体导航。

## 当前待验项

### 1. 启动与数据兼容

- [ ] 应用启动后，任务列表可以正常读取既有数据。
- [ ] 历史任务在升级后默认补齐为 `中` 优先级。
- [ ] 升级后的列表不会因为缺少 `priority` 而报错或空白。

### 2. 优先级模型

- [ ] 新建任务时可以设置优先级。
- [ ] 编辑任务时可以修改优先级。
- [ ] `紧急 / 高 / 中 / 低` 四档优先级在界面上显示正确。
- [ ] 优先级在重启后仍能保留。

### 3. 更多条件

- [ ] 左侧 `更多条件` 中可以看到任务组、优先级、时间范围和排序方式。
- [ ] 任务组筛选可正常工作，并与状态筛选组合生效。
- [ ] 优先级筛选可正常工作，并与状态筛选组合生效。
- [ ] 时间范围筛选可正常工作，并与其他条件组合生效。
- [ ] 切换排序方式后，列表顺序会同步更新。

### 4. 列表排序与展示

- [ ] 默认列表在同一批任务数据下与文档约定的排序结果、状态标识和空状态文案一致。
- [ ] 每条任务至少显示一个可区分 `紧急 / 高 / 中 / 低` 的优先级标识。
- [ ] 完成态、进行中态和选中态在同一列表中可被稳定区分，不出现状态样式混用。
- [ ] 状态筛选与更多条件叠加后，刷新或重开应用不会导致结果口径漂移。

### 5. 批量操作

- [ ] 列表可以进入批量模式。
- [ ] 进入批量模式后，界面显示已选数量和批量操作栏。
- [ ] 选中多条任务后，可以一次性修改其优先级。
- [ ] 选中多条任务后，可以一次性修改其所属任务组。
- [ ] 选中多条任务后，可以一次性标记完成。
- [ ] 批量模式下不存在批量删除入口或批量删除执行路径。
- [ ] 退出批量模式后，已选中状态被清空，界面恢复普通浏览态。

### 6. 任务概览

- [ ] 可以从顶部入口打开 `任务概览` 弹窗。
- [ ] 概览中显示 `紧急 / 高 / 中 / 低` 四档优先级的数量或占比。
- [ ] 概览中显示最近 7 天趋势区域。
- [ ] 概览中显示本周周报卡片区域。
- [ ] 首次打开概览时，趋势图默认展示完成数。
- [ ] 概览提供切换到“新建 / 完成双线”展示的入口，切换后视图同步更新。

### 7. 快捷操作强化

- [ ] 顶部全局区域同时保留 `任务概览`、`任务组管理` 与 `新建` 三个入口。
- [ ] 列表区域存在进入和退出批量模式的直接入口。
- [ ] 列表区域存在对单条任务直接标记完成和进入编辑的入口。
- [ ] 单条任务至少提供一条不进入复杂流程即可切换优先级的操作路径。
- [ ] 界面内不存在新的复杂快捷键体系说明、入口或配置项。

### 8. 界面精修

- [ ] 标题、正文和辅助文本在同一页面内使用可区分且一致的字号层级。
- [ ] 相邻按钮、输入框和切换控件的间距与高度不存在明显不一致。
- [ ] `更多条件` 区域中的布局、标签和选中态表达在各类条件项之间保持一致。
- [ ] 概览弹窗在重复打开、关闭和切换内容后仍保持固定的顶部、内容区和卡片区结构。

### 9. 构建验证

- [ ] `tsc -b` 通过。
- [ ] `cargo check` 通过。
