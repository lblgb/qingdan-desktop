# `v0.20` 验收清单

## 文件说明

本文档用于沉淀 `v0.20` 的统一手工验收口径，作为本轮实施、回归和收口的唯一执行清单。

## 验收范围

- 任务优先级正式落地并可持久化。
- 左侧 `更多条件` 扩展到任务组、优先级、时间范围和排序方式。
- 列表支持轻量批量操作。
- `任务概览` 保持弹窗形态并增强统计内容。
- 主界面保持当前产品骨架，不新增独立概览页或重做整体导航。

## 当前待验项

### 1. 启动与数据兼容

- [ ] 应用启动后，任务列表可以正常读取既有数据。
- [ ] 历史任务在升级后默认补齐为 `中` 优先级。
- [ ] 升级后的列表不会因为缺少 `priority` 而报错或空白。

### 2. 优先级模型

- [ ] 新建任务时可以设置优先级。
- [ ] 编辑任务时可以修改优先级。
- [ ] `紧急 / 高 / 中 / 低` 四档优先级在界面上显示正确。
- [ ] 优先级在重启后仍能保留。

### 3. 更多条件

- [ ] 左侧 `更多条件` 中可以看到任务组、优先级、时间范围和排序方式。
- [ ] 任务组筛选可正常工作，并与状态筛选组合生效。
- [ ] 优先级筛选可正常工作，并与状态筛选组合生效。
- [ ] 时间范围筛选可正常工作，并与其他条件组合生效。
- [ ] 切换排序方式后，列表顺序会同步更新。

### 4. 列表排序与展示

- [ ] 默认列表展示口径与文档约定一致。
- [ ] 优先级状态在列表中有清晰可见的标识。
- [ ] 完成态、进行中态和选中态的表达不会互相混淆。
- [ ] 状态筛选与更多条件叠加后，结果仍然稳定。

### 5. 批量操作

- [ ] 列表可以进入批量模式。
- [ ] 批量模式下能看到已选数量与批量操作栏。
- [ ] 可以批量修改优先级。
- [ ] 可以批量归组。
- [ ] 可以批量标记完成。
- [ ] 本版不支持批量删除。
- [ ] 退出批量模式后，界面恢复普通浏览态。

### 6. 任务概览

- [ ] 可以从顶部入口打开 `任务概览` 弹窗。
- [ ] 概览中可以看到优先级分布。
- [ ] 概览中可以看到最近 7 天趋势。
- [ ] 概览中可以看到本周周报卡片。
- [ ] 趋势默认显示完成数。
- [ ] 概览支持切换到新建/完成双线展示。

### 7. 界面精修

- [ ] 字体层级比当前基线更清晰。
- [ ] 按钮密度与节奏保持统一。
- [ ] 更多条件的结构与选中态表达保持一致。
- [ ] 概览弹窗呈现为稳定的信息面板风格。

### 8. 构建验证

- [ ] `tsc -b` 通过。
- [ ] `cargo check` 通过。

## 当前明确不纳入

- 快速新建条。
- 标签系统。
- 多任务组归属。
- 子组或嵌套组。
- 拖拽排序。
- 批量删除。
- 日历主视图。
- 服务端同步。
- 多人协作。
- 复杂快捷键体系。
- 独立的排序偏好管理面板。

## 建议执行顺序

1. 先确认优先级数据补齐与历史数据兼容。
2. 再验证更多条件与排序方式的联动。
3. 然后验证批量模式与概览增强。
4. 最后统一做一次界面精修和构建检查。
