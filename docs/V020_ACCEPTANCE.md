# `v0.20` 验收清单

## 文件说明

本文档用于沉淀 `v0.20` 的统一手工验收口径，作为本轮实施、回归和收口时的唯一执行清单。

## 当前状态

- 当前版本目标已按 `v0.20` 设计范围实现完毕。
- 本清单已根据当前代码状态更新为“已实现基线”。
- 自动验证已完成：
  - `cmd /c npx.cmd vitest run`
  - `cmd /c node_modules\\.bin\\tsc.cmd -b`
  - `cargo check`
  - `cargo test query_tasks_filters_active_urgent_tasks_when_sorted_by_priority -- --nocapture`
  - `cargo test init_database_adds_priority_column_to_legacy_tasks_table -- --nocapture`

## 验收范围

- 任务优先级正式落地并可持久化。
- 左侧 `更多条件` 扩展到任务组、优先级、时间范围和排序方式。
- 列表支持轻量批量操作。
- `任务概览` 保持弹窗形态并增强统计内容。
- 快捷操作强化覆盖全局、列表和任务级高频动作。
- 主界面保持当前产品骨架，不新增独立概览页或重做整体导航。

## 当前验收项

### 1. 启动与数据兼容

- [x] 应用启动后，任务列表可以正常读取既有数据。
- [x] 历史任务在升级后默认补齐为 `中` 优先级。
- [x] 升级后的列表不会因为缺少 `priority` 而报错或空白。

### 2. 优先级模型

- [x] 新建任务时可以设置优先级。
- [x] 编辑任务时可以修改优先级。
- [x] `紧急 / 高 / 中 / 低` 四档优先级在界面上显示正确。
- [x] 优先级在重启后仍能保留。

### 3. 更多条件

- [x] 左侧 `更多条件` 中可以看到任务组、优先级、时间范围和排序方式。
- [x] 任务组筛选可正常工作，并与状态筛选组合生效。
- [x] 优先级筛选可正常工作，并与状态筛选组合生效。
- [x] 时间范围筛选可正常工作，并与其他条件组合生效。
- [x] 切换排序方式后，列表顺序会同步更新。

### 4. 列表排序与展示

- [x] 默认列表在同一批任务数据下与文档约定的排序结果、状态标识和空状态文案一致。
- [x] 每条任务至少显示一个可区分 `紧急 / 高 / 中 / 低` 的优先级标识。
- [x] 完成态、进行中态和选中态在同一列表中可被稳定区分，不出现状态样式混用。
- [x] 状态筛选与更多条件叠加后，不会出现结果口径漂移。

### 5. 批量操作

- [x] 列表可以进入批量模式。
- [x] 进入批量模式后，界面显示已选数量和批量操作栏。
- [x] 选中多条任务后，可以一次性修改其优先级。
- [x] 选中多条任务后，可以一次性修改其所属任务组。
- [x] 选中多条任务后，可以一次性标记完成。
- [x] 批量模式下不存在批量删除入口或批量删除执行路径。
- [x] 退出批量模式后，已选中状态被清空，界面恢复普通浏览态。

### 6. 任务概览

- [x] 可以从顶部入口打开 `任务概览` 弹窗。
- [x] 概览中显示 `紧急 / 高 / 中 / 低` 四档优先级的数量或占比。
- [x] 概览中显示最近 7 天趋势区域。
- [x] 概览中显示本周摘要卡片区域。
- [x] 首次打开概览时，趋势图默认显示完成数。
- [x] 概览提供切换到“新增 / 完成双线”展示的入口，切换后视图同步更新。

### 7. 快捷操作强化

- [x] 顶部全局区域同时保留 `任务概览`、`任务组管理` 和 `新建` 三个入口。
- [x] 列表区域存在进入和退出批量模式的直接入口。
- [x] 列表区域存在对单条任务直接标记完成和进入编辑的入口。
- [x] 单条任务至少提供一条不进入复杂流程即可切换优先级的操作路径。
- [x] 界面内不存在新的复杂快捷键体系说明、入口或配置页。

### 8. 界面精修

- [x] 相邻按钮、输入框和切换控件的间距与高度不存在明显不一致。
- [x] `更多条件` 区域中的布局、标签和选中态表达在各类条件项之间保持一致。
- [x] 概览弹窗在重复打开、关闭和切换内容后仍保持固定的顶部、内容区和卡片区结构。
