# 第二版验收清单

## 文件说明

本文档用于沉淀 `v0.1.5` 在当前阶段的手工验收口径，作为第二版收口、问题回归和是否进入下一阶段的统一清单。

## 验收范围

- 任务组数据层与前端状态层协同工作。
- 新建任务已切换为按钮触发的弹窗录入流。
- 状态筛选与任务组筛选可同时工作。
- 任务组管理具备最小闭环能力。
- 现有第一版的时间排序、时间分组和基础反馈语义未被第二版破坏。

## 当前待验证项

### 1. 启动与数据初始化

- [ ] 应用启动后任务列表与任务组列表都能成功读取。
- [ ] 若本地旧数据库中原本没有 `group_id` 字段，启动后应用仍可正常进入主界面。

### 2. 任务组基础链路

- [ ] 可以在“新建任务”弹窗内创建任务组。
- [ ] 新建任务组成功后，弹窗会自动选中刚创建的任务组。
- [ ] 可以在任务组管理面板中编辑任务组名称与说明。
- [ ] 可以在任务组管理面板中删除任务组。
- [ ] 删除任务组后，原组内任务自动变为未分组。

### 3. 任务与任务组关系

- [ ] 新建任务时可以直接选择所属任务组。
- [ ] 新建后任务会在列表中展示正确的任务组归属。
- [ ] 在任务列表编辑态中可以修改所属任务组。
- [ ] 任务移回未分组后，列表中的任务组信息同步更新。

### 4. 筛选与展示语义

- [ ] `全部任务 / 进行中 / 已完成` 状态筛选仍可正常工作。
- [ ] `全部组 / 未分组 / 某个具体组` 组筛选可正常工作。
- [ ] 状态筛选与组筛选叠加后，列表结果符合两者交集。
- [ ] 组筛选生效时，任务列表仍按现有时间语义分组展示。
- [ ] 当前筛选中的任务组会在管理面板中正确高亮。

### 5. 弹窗交互与反馈

- [ ] 点击“新建任务”入口后会打开弹窗。
- [ ] `Esc` 可关闭弹窗。
- [ ] 点击遮罩可关闭弹窗。
- [ ] 新建任务过程中可看到保存中反馈。
- [ ] 新建任务组过程中可看到处理中反馈。
- [ ] 关闭弹窗后再次打开，表单应回到干净状态。

### 6. 构建验证

- [ ] `tsc -b` 通过。
- [ ] `cargo check` 通过。

## 当前已知非阻塞项

- 当前还没有拖拽排序、自定义组排序位和批量整理能力。
- 当前还没有“未保存内容离开提醒”。
- Python 与 skill 脚本环境仍未恢复，计划在第二版收口后单独处理。

## 建议执行顺序

1. 先验任务组创建、编辑、删除和删除后任务回退未分组。
2. 再验任务新建弹窗与任务归组链路。
3. 最后验状态筛选、组筛选和时间分组是否共同成立。
