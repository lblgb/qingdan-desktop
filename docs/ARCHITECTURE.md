## `v0.50.0` 当前实施基线

### 版本目标

- 在不推翻当前主工作台信息架构的前提下，把界面语言升级为深色科技控制台风格，并补齐备份、恢复、导出与搜索定位能力。
- 继续保持当前 `taskStore + AppShell + TaskList + TaskOverview + TaskSettings` 架构稳定，不引入新的重型主页面。

### 当前纳入范围

- 主工作台、列表、概览、设置、提醒、备份与任务详情统一到深色科技控制台视觉语言。
- 右上角补齐 `设置 / 提醒 / 备份` 系统图标入口，并让提醒入口展示待处理数量角标。
- 建立本地数据安全闭环，至少包含手动备份、备份恢复、恢复前风险确认与错误反馈。
- 补齐 JSON / CSV 数据导出能力，支持全量导出与当前筛选结果导出。
- 补齐按标题与备注检索的全局搜索，并支持搜索后自动定位与高亮目标任务。

### 当前明确不包含

- 云同步或多端同步。
- 日历主视图、子任务系统或标签系统重构。
- 自动定时备份策略。
- 复杂搜索语法、命令面板系统或 CSV 导入。
- PDF、图片等展示型导出。

## `v0.40.0` 历史实施基线

### 版本目标

- 在不重做主工作台的前提下，收口提醒系统、补齐任务详情与备注、完善完成与归档语义，并让复盘与质量提示具备可验收的闭环。
- 保持当前 `taskStore + AppShell + TaskList + TaskOverview + TaskSettings` 架构稳定，不引入新的重型主页面。

### 当前纳入范围

- 提醒系统只保留运行期间的桌面通知能力，设置弹窗必须明确通知权限状态与测试通知结果反馈。
- 任务完成态写入 `completed_at`，改回进行中时清空 `completed_at`，并通过归档语义与删除语义严格分离。
- 任务详情弹窗负责标题、备注、优先级、任务组和截止日期的轻量编辑，并展示创建、更新、完成和归档时间。
- 列表与概览中的备注只显示摘要或标识，不在列表层展开完整备注内容。
- 任务概览补齐最近完成任务、本周复盘卡片、月度完成趋势和任务质量提示。

### 当前明确不包含

- 后台常驻或脱离应用运行期的提醒调度。
- 归档后删除、取消归档的完整生命周期设计。
- 重型独立详情页、日历页或复杂复盘工作流。

## `v0.30.0` 历史实施基线

### 版本目标

- 在 `v0.20.0` 基线上完成筛选恢复默认、低侵入成功反馈、统一失败弹窗和全局提醒系统。
- 保持当前任务工作台结构稳定，不新增重型主页面，不把产品带向日历系统或复杂规则引擎。

### 当前纳入范围

- `更多条件` 面板内新增 `恢复默认筛选`，一键恢复为：
  - `全部任务`
  - `全部组`
  - `全部优先级`
  - `全部时间`
  - `默认推荐`
- 成功反馈统一改为右下角轻提示浮层，不再占用列表内容区。
- 失败反馈统一改为轻量错误弹窗，当前只提供关闭动作。
- 右上角新增 `设置` 入口，集中管理提醒偏好。
- 右上角新增 `铃铛` 入口，承接提醒详情弹窗。
- 首页列表上方新增轻量关注条，仅在存在提醒内容时显示。
- 提醒规则改为全局配置，不对单个任务做独立提醒设置。
- 桌面系统通知只对“有截止日期且达到优先级门槛”的任务生效。
- 高优先级但无截止日期的任务只进入应用内关注集合，不发送系统通知。
- 点击提醒项后，复用统一定位状态，将用户带回对应任务，自动滚动并短暂高亮。

### 当前提醒语义

- 提醒优先级门槛固定为：
  - `紧急及以上`
  - `高及以上`
  - `中及以上`
- 提醒时间固定为：
  - `准时`
  - `提前 10 分钟`
  - `提前 1 小时`
  - `提前 1 天`
  - `自定义时间`
- 应用内提醒与桌面通知都遵循同一套优先级门槛与时间窗口定义。

### 本轮明确不包含

- 保存用户自定义默认视图
- 每任务单独提醒配置
- 多提醒点
- 重复提醒
- 日历页或周历页
- 删除相关行为改造

## 技术栈

- 桌面框架：Tauri
- 前端框架：React
- 语言：TypeScript
- 构建工具：Vite
- 状态管理：Zustand
- 数据校验：Zod
- 时间处理：dayjs
- 本地数据库：SQLite

## 设计目标

- 保持本地优先的桌面待办工具定位。
- 继续沿用 `v0.1.5` 已收口的主界面骨架，不做产品形态推翻。
- 通过 Tauri 命令边界统一收敛宿主能力与数据库访问。
- 让筛选、排序、批量操作和概览统计共享同一套任务语义，避免口径漂移。

## 模块边界

### 前端

- `src/app`
  - 应用入口、全局布局、页面装配和顶层容器。
- `src/components`
  - 通用组件，例如按钮、输入框、弹窗、空状态、图表容器和列表容器。
- `src/features/tasks`
  - 任务领域模块。
  - 包含任务列表、任务表单、任务筛选、任务概览派生数据和前端任务类型。
- `src/lib`
  - 时间工具、Tauri API 封装、通用常量和轻量辅助函数。
- `src/stores`
  - Zustand 状态管理。
  - 负责筛选条件、排序方式、批量模式和概览开关等界面状态编排。
- `src/types`
  - 前端共享类型定义。

### Tauri / SQLite

- `src-tauri/src/commands`
  - 对前端暴露的命令入口。
  - 负责参数校验、调用数据库访问层并返回业务所需最小结果。
- `src-tauri/src/db`
  - SQLite 连接、建表、迁移与查询封装。
- `src-tauri/src/models`
  - Rust 侧数据结构、序列化模型和命令层共用枚举。

## `v0.20` 规划范围

### 版本目标

- 在不推翻 `v0.1.5` 主界面骨架的前提下，补齐优先级、更多条件、轻量批量操作、任务概览增强和系统级界面精修。
- 补齐 `快捷操作强化`，缩短全局入口、列表动作和任务级轻量操作的路径。
- 保持“先写清文档基线，再按文档实现”的推进方式。

### 本轮纳入范围

#### 1. 优先级正式落地

- 为任务新增正式字段 `priority`。
- 固定四档优先级：
  - `urgent` -> `紧急`
  - `high` -> `高`
  - `medium` -> `中`
  - `low` -> `低`
- 历史任务在数据库升级后默认补为 `medium`。
- 命令层、前端展示、排序和统计必须共享同一套优先级语义。

#### 2. 更多条件体系扩展

- 左侧 `更多条件` 扩展为四类：
  - `任务组`
  - `优先级`
  - `时间范围`
  - `排序方式`
- 主筛选仍只负责：
  - `全部任务`
  - `进行中`
  - `已完成`
- 最终结果定义为交集：先由主筛选限定状态集合，再由更多条件继续缩小结果集。

#### 2.1 枚举口径

- `任务组`：
  - `全部组`
  - `未分组`
  - `某个具体组`
- `优先级`：
  - `全部优先级`
  - `紧急`
  - `高`
  - `中`
  - `低`
- `时间范围`：
  - `全部时间`
  - `今天`
  - `未来 7 天`
  - `已逾期`
  - `无日期`
- `排序方式`：
  - `默认推荐`
  - `截止日期优先`
  - `优先级优先`
  - `最近更新`

#### 2.2 排序语义

- `默认推荐` 延续当前时间语义：
  - 进行中优先于已完成
  - 有截止日期优先于无截止日期
  - 截止日期更早的优先
  - 截止日期相同或无截止日期时，按最近更新时间倒序
- `截止日期优先` 更强调时间驱动。
- `优先级优先` 先按 `紧急 / 高 / 中 / 低` 排序，再套时间语义规则。
- `最近更新` 以最近更新时间为主。

#### 2.3 本轮限制

- 只支持固定排序方式切换。
- 不支持用户自定义排序规则组合。
- 不支持独立的排序偏好管理面板。

#### 3. 轻量批量操作

- 提供列表多选模式。
- 默认列表不常驻复选框。
- 进入批量模式后，顶部工具条切换为批量栏。
- 本轮允许：
  - 批量改优先级
  - 批量归组
  - 批量标记完成
- 本轮明确不包含批量删除，也不包含批量编辑标题、备注和截止日期。

#### 4. 任务概览增强

- `任务概览` 继续以弹窗承载，不新增独立概览页面。
- 概览增强重点包含：
  - 优先级分布
  - 最近 7 天趋势
  - 本周周报卡片
- 最近 7 天趋势默认展示完成数，并支持切换为“新增 / 完成双线”对比视图。
- 本周周报卡片至少展示：本周完成数、本周新增数、逾期变化、当前最高优先级未完成数。

#### 5. 系统级界面精修

- 优化字体层级。
- 优化按钮密度与节奏。
- 统一更多条件的结构与选中态表达。
- 强化列表状态表达。
- 将概览弹窗收敛为更稳定的信息面板风格。

#### 6. 快捷操作强化

- 全局层保留 `任务概览`、`任务组管理` 和 `新建` 三个高频入口。
- 列表层保留进入或退出批量模式、标记完成和进入编辑的轻量动作。
- 任务层保留轻量切换优先级的快捷路径。
- 本轮只强化高频动作路径，不引入复杂快捷键体系。

### 责任边界

- 命令层负责优先级补齐、更多条件查询口径、批量动作所需最小数据接口，以及旧数据兼容升级。
- 前端状态仓库负责筛选条件、排序方式、批量模式和概览开关的统一编排。
- 前端列表负责展示优先级、更多条件结果、批量选择态和任务概览入口，不把复杂查询语义散落到页面组件里。
- 概览依然是派生数据视图，和主列表共享同一套优先级与时间范围定义。

### 本轮明确不包含

- 快速新建条
- 标签系统
- 多任务组归属
- 子组或嵌套组
- 拖拽排序
- 批量删除
- 日历主视图
- 服务端同步
- 多人协作
- 用户自定义多字段排序组合
- 独立的排序偏好管理面板
- 复杂快捷键体系

### 实施顺序

1. 先更新架构文档与验收文档，定清 `v0.20` 语义基线。
2. 再补 SQLite 与命令层的数据模型、兼容迁移和查询参数。
3. 再扩前端状态仓库，统一筛选、排序和批量模式状态。
4. 再做列表中的优先级展示、更多条件、排序切换与批量操作。
5. 最后补任务概览增强与系统级界面精修。

### 风险控制

- 旧库必须安全补齐 `priority`，不能依赖删库。
- 前端、命令层、概览统计必须共享同一套优先级与时间范围定义。
- 严格保持“不做快速新建条、不做批量删除、不做复杂快捷键体系”的边界。
- 默认态保持克制，把复杂度放到进入筛选、概览或批量模式之后再暴露。

### 验收与收口

- `docs/V020_ACCEPTANCE.md` 是 `v0.20` 唯一的手工验收执行清单。
- 本文档只维护范围、责任边界、实施顺序与风险控制，不重复维护第二份验收条目。
- 若 `v0.20` 收口完成，应新增独立收口文档，不在本文档中追加历史实现流水账。
## 当前实施落地要点

- `priority` 已贯通 SQLite、Tauri 命令层、前端类型、列表展示、新建与编辑入口。
- 左侧 `更多条件` 已正式扩展为 `任务组 / 优先级 / 时间范围 / 排序方式` 四类条件面板，并统一接入同一套查询状态。
- 列表已支持多选批量模式，当前只提供批量改优先级、批量归组和批量完成，不提供批量删除。
- 概览弹窗已落地优先级分布、最近 7 天趋势和本周摘要卡片，趋势默认展示完成数，并支持切换到“新增 / 完成”双线。
