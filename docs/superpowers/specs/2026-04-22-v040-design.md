# `v0.40.0` 设计规格

## 1. 版本定位

`v0.40.0` 定义为“稳健增强版”。本版本不重做主工作台，不新增重型主页面，而是在 `v0.30.0` 的基础上补齐三条主线：

- 提醒系统收口：修正桌面通知能力表达，补通知权限状态和测试通知入口。
- 完成与复盘增强：让已完成任务可回看、可归档、可形成轻量复盘。
- 任务详情与整理轻量增强：补备注、详情弹窗、低质量提醒和少量整理动作。

本版本继续保持本地优先、桌面待办、低复杂度工作台定位。

## 2. 纳入范围

### 2.1 提醒系统收口

- 保留 `v0.30.0` 已接入的运行时桌面系统通知能力。
- 设置弹窗中修正桌面通知说明，不再出现“仅保留偏好设置，不会实际调度系统通知”这类旧口径。
- 设置弹窗展示通知权限状态：
  - `已允许`
  - `未授权`
  - `被系统拒绝`
  - `检测失败`
- 设置弹窗新增 `发送测试通知` 按钮。
- 明确说明桌面通知仅在轻单运行期间触发，不做后台守护。
- 测试通知成功后走右下角轻提示。
- 测试通知失败后走统一错误弹窗。
- 应用内提醒不依赖系统通知权限，继续独立工作。

### 2.2 完成与复盘增强

- 任务完成时写入完成时间。
- 最近完成任务可在概览中查看。
- 新增归档语义：已完成任务可以归档，归档不等于删除。
- 归档任务默认从普通工作列表中隐藏。
- 新增归档视图，用于查看已归档任务。
- 任务概览中新增复盘区块：
  - 最近完成
  - 本周复盘卡片
  - 月度完成趋势
  - 任务质量提示

### 2.3 任务详情与整理轻量增强

- 新增任务详情弹窗。
- 任务新增备注字段。
- 详情弹窗中可查看并编辑：
  - 标题
  - 备注
  - 优先级
  - 任务组
  - 截止日期
- 详情弹窗中展示：
  - 创建时间
  - 更新时间
  - 完成时间
  - 归档时间
- 列表中只展示备注标识或一行摘要，不展示完整备注。
- 新建弹窗暂不加入备注输入，避免新建流程变重。
- 新建和编辑标题时提供低质量提醒：
  - 空标题继续禁止提交。
  - 过短标题只提醒，不阻断。
  - 重复标题只提醒，不阻断。
- 高优先级无备注任务在详情弹窗和复盘卡片中提示，不强制填写。

## 3. 信息架构

### 3.1 顶部动作区

顶部动作区保持克制，不新增大量主按钮。继续沿用现有入口：

- `任务概览`
- `任务组管理`
- `新建`
- `设置`
- `铃铛`

复盘能力优先放入 `任务概览`，不新增独立顶层 `复盘` 入口。

### 3.2 左侧工作视图

左侧工作视图在现有主筛选基础上承接归档入口。建议方案：

- 主筛选继续保持：
  - `全部任务`
  - `进行中`
  - `已完成`
- 新增轻量 `归档` 入口，用于查看已归档任务。

归档入口可以作为工作视图的一项，也可以放在主筛选下方的次级位置。实现时以不破坏现有筛选节奏为准。

### 3.3 右侧任务列表

- 任务标题或详情按钮可打开任务详情弹窗。
- 已归档任务不出现在普通列表中。
- 归档视图只展示已归档任务。
- 列表中只用轻量方式表达备注存在，不把完整备注展开到列表。

### 3.4 设置弹窗

设置弹窗继续承接提醒配置，并补齐可信状态：

- 应用内提醒开关
- 桌面系统通知开关
- 通知权限状态
- 发送测试通知
- 提醒优先级门槛
- 提醒时间
- 运行时通知说明

### 3.5 任务概览

任务概览继续以弹窗承载，并新增复盘区块：

- 最近完成任务列表：展示最近 5-10 条完成任务。
- 本周复盘卡片：本周完成数、归档数、最高优先级完成数、仍逾期未完成数。
- 月度趋势：按周或按天展示本月完成节奏。
- 任务质量提示：过短标题数、重复标题候选数、高优先级无备注任务数。

## 4. 数据模型与状态语义

### 4.1 新增字段

任务表新增或明确以下字段：

- `note`：任务备注，默认空字符串或 `null`。
- `completed_at`：完成时间，任务从进行中变为已完成时写入。
- `archived_at`：归档时间，用户执行归档动作时写入。

### 4.2 状态分层

- `status` 只表示任务是否完成：`active / completed`。
- `completed_at` 表示最近一次完成时间。
- `archived_at` 表示任务是否已从日常列表中收起。
- `note` 只作为任务详情补充信息，不参与主状态。

### 4.3 完成时间口径

- 任务被标记完成时写入 `completed_at`。
- 已完成任务被改回进行中时清空 `completed_at`。
- 再次完成时重新写入新的 `completed_at`。
- 历史多次完成记录不进入 `v0.40.0`。

### 4.4 归档口径

- 只有已完成任务可以归档。
- 归档不等于删除。
- 归档后任务默认不出现在普通工作列表中。
- 归档任务仍参与完成趋势和复盘统计。
- `v0.40.0` 默认不提供取消归档，避免归档语义变复杂。

## 5. 提醒系统口径

桌面通知在 `v0.40.0` 的目标是“可信可验证”，不是后台提醒系统。

- 开启桌面系统通知后，应用运行期间会按全局提醒规则尝试发送系统通知。
- 如果系统权限未授权，保存设置或发送测试通知时可以触发权限请求。
- 如果系统权限被拒绝，应提示用户到系统设置中开启通知权限。
- 如果通知发送失败，应使用统一错误弹窗反馈。
- 不做后台守护。
- 不做通知历史。
- 不做重复提醒。
- 不做每任务单独提醒配置。

## 6. 整理动作

### 6.1 单任务归档

- 入口放在已完成任务详情弹窗中。
- 进行中任务不展示归档动作。

### 6.2 批量归档

- 批量模式中新增批量归档动作。
- 只有当选中项全部为已完成任务时，批量归档可用。
- 如果选中项混有进行中任务，批量归档按钮禁用，并给出原因提示。

### 6.3 低质量提醒

- 空标题继续阻断提交。
- 过短标题只提示，不阻断保存。
- 重复标题只提示，不阻断保存。
- 不做自动合并。
- 不做自动删除。

## 7. 明确不做

`v0.40.0` 明确不包含：

- 后台常驻通知
- 通知历史中心
- 自动删除任务
- 多提醒规则
- 每任务单独提醒配置
- 历史多次完成记录
- 复杂报表
- 多任务组
- 标签系统
- 云同步或多端
- 删除相关行为改造

## 8. 验收基线

- 设置弹窗不再误导桌面通知能力。
- 设置弹窗可查看通知权限状态。
- 设置弹窗可发送测试通知。
- 测试通知成功后显示右下角轻提示。
- 测试通知失败后显示统一错误弹窗。
- 任务可编辑备注。
- 任务详情弹窗可展示标题、备注、优先级、任务组、截止日期和关键时间。
- 任务完成时写入 `completed_at`。
- 已完成任务可单独归档。
- 批量模式可对全已完成选择集执行批量归档。
- 归档任务默认从普通列表隐藏。
- 归档视图可查看归档任务。
- 任务概览可展示最近完成、本周复盘、月度趋势和任务质量提示。
- 过短标题和重复标题只提醒，不阻断保存。
- 高优先级无备注任务可在详情或复盘中被提示。
- 不出现自动删除、后台守护、通知历史或多提醒规则。
- `cmd /c npx.cmd vitest run --exclude=.worktrees/**` 通过。
- `cmd /c npx.cmd tsc -b` 通过。
- `cargo check` 通过。

5. 实现完成时间、归档视图和批量归档。
