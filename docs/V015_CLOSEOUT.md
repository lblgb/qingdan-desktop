# v0.1.5 收口说明

## 文件说明

本文档用于确认 `v0.1.5` 的最终收口范围、已完成内容、明确不包含项，以及进入下一阶段前的版本基线。

## 版本结论

- `v0.1.5` 已确认收口。
- 当前版本目标聚焦于“任务组织能力升级”和“主界面交互收束”。
- 当前代码、文档和 GitHub 远端已同步到同一基线，可作为下一阶段迭代起点。

## 本版完成项

- 顶部全局动作区已经稳定为：
  - `任务概览`
  - `任务组管理`
  - `新建`
- `新建任务` 已改为顶部入口 + 弹窗录入流。
- 任务组已具备完整基础能力：
  - 新建
  - 编辑
  - 删除
  - 任务归组
  - 删除组后任务自动转回未分组
- 左侧筛选已收敛为：
  - 主筛选：`全部任务 / 进行中 / 已完成`
  - 附加筛选：`更多条件 -> 任务组 -> 全部组 / 未分组 / 某个组`
- `任务组管理` 已从页面主体移出，改为顶部独立入口和弹窗。
- `任务概览` 已具备独立弹窗，并支持：
  - 汇总指标
  - 时间状态分布
  - 任务组分布
- 任务清单已补充分页：
  - 默认每页 10 项
  - 支持页码和上下页切换
  - 筛选变化后自动回到第一页
- 界面配色与层级已完成一轮统一收束，当前主界面风格基线已稳定。

## 当前版本保留的产品语义

- 任务仍然按既有时间语义排序与分组。
- 任务组属于附加属性，不替代主任务状态。
- 一个任务在当前版本中最多只属于一个任务组。
- 本地持久化链路仍以 `Tauri + SQLite` 为正式实现。

## 明确不包含项

- 优先级正式模型与优先级筛选
- 标签系统
- 多任务组归属
- 子组/嵌套组
- 拖拽排序
- 任务批量操作
- 趋势图、周报、统计历史
- 服务端同步和多人协作

## 当前建议的下一阶段方向

- 在附加筛选层继续扩：
  - 优先级
  - 时间范围
  - 排序方式
- 继续收界面细节：
  - 字号层级
  - 按钮密度
  - 留白节奏
- 如果进入下一个小版本，建议优先做“优先级 + 更多条件体系扩展”。

## 版本基线说明

- 本版收口时，工作日志、架构文档和代码实现已保持同步。
- 后续继续推进时，应默认以本文档和 `docs/WORKLOG.md` 作为版本边界参考。
