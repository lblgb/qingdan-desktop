# `v0.30.0` 验收清单

## 说明

本文档是 `v0.30.0` 当前唯一有效的手工验收基线。
如实现边界发生变化，应先更新本文档和 `docs/ARCHITECTURE.md`，再继续开发或收口。

## 范围总览

- 筛选体验收口
- 成功/失败反馈重构
- 全局提醒系统
- 提醒到任务的定位闭环

## 手工验收项

### 1. 恢复默认筛选

- 打开 `更多条件` 面板后，可看到 `恢复默认筛选` 入口。
- 点击后，主筛选恢复为 `全部任务`。
- 点击后，任务组恢复为 `全部组`。
- 点击后，优先级恢复为 `全部优先级`。
- 点击后，时间范围恢复为 `全部时间`。
- 点击后，排序方式恢复为 `默认推荐`。
- 点击后，如当前不在第 1 页，分页自动回到第 1 页。

### 2. 成功反馈

- 新建任务成功后，不再在列表内容区显示结果提示。
- 修改任务优先级成功后，不再在列表内容区显示结果提示。
- 批量改优先级 / 归组 / 完成成功后，不再在列表内容区显示结果提示。
- 成功反馈统一显示为右下角轻提示，并会自动消失。

### 3. 失败反馈

- 新建任务失败时，显示统一错误弹窗。
- 保存提醒设置失败时，显示统一错误弹窗。
- 错误弹窗至少显示标题、失败原因和关闭动作。
- 失败信息不再直接挤占任务列表区域。

### 4. 提醒设置

- 顶部动作区新增 `设置` 入口。
- 设置弹窗内可开关应用内提醒。
- 设置弹窗内可开关桌面系统通知。
- 设置弹窗内可设置提醒优先级门槛：`紧急及以上 / 高及以上 / 中及以上`。
- 设置弹窗内可设置提醒时间：`准时 / 提前 10 分钟 / 1 小时 / 1 天 / 自定义时间`。

### 5. 应用内提醒

- 当存在即将到期、已逾期或高优先级未排期任务时，列表上方出现轻量关注条。
- 顶部动作区新增 `铃铛` 入口。
- 铃铛弹窗可按分组展示提醒项：
  - `即将到期`
  - `已逾期`
  - `高优先级但无日期`
  - `最近已提醒`

### 6. 桌面通知

- 开启桌面通知后，符合门槛且有截止日期的任务会在应用运行时触发系统通知。
- 无截止日期的高优先级任务不会触发系统通知。
- 同一提醒项在自动刷新过程中不会重复刷屏。

### 7. 提醒定位闭环

- 点击提醒弹窗中的某条提醒后，提醒弹窗关闭。
- 列表能定位到对应任务时，会自动滚动到任务位置。
- 被定位到的任务会短暂高亮。
- 高亮结束后，提醒定位状态会自动清除。
